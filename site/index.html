How it works

Three pieces, one encrypted channel. The relay routes frames; only the PWA and your laptop hold keys.

Browser
PWA

React app on your phone. Holds half the key.

Blind
Relay

Routes encrypted frames and stores an encrypted mailbox. No keys, no plaintext, no user identifiers. Self-hostable.

Daemon
Agent

Runs on your laptop. Owns NaCl keys, runs coding agent + git.

The relay never sees your code

The relay routes WebSocket frames and holds an encrypted mailbox for syncing between your devices — but it has no keys. The PWA on your phone and the agent on your laptop are the only parties that can read what's exchanged; everything in transit and at rest is sealed with NaCl box encryption. What little metadata the relay does keep contains no information that ties back to a real person or organization — not for us, not for anyone running the server.

What the relay sees
  • ·Encrypted byte frames in transit
  • ·Encrypted blobs at rest (device-to-device mailbox)
  • ·Random session IDs — no usernames, no org names, no emails
  • ·Web Push fan-out targets (opaque tokens)
  • ·Connection lifecycle events
What stays on your devices
  • ·Source code, diffs, file contents
  • ·Coding-agent prompts and output
  • ·Commit messages and git history
  • ·Repo paths and machine names

Don't want to trust the public relay either? It's a small Node server with a Dockerfile in the repo — run your own and point your PWA at it.

End-to-end encrypted

NaCl keypair handshake between PWA and agent. The relay routes frames and stores encrypted blobs — it has no keys, so it never sees plaintext.

Remote control your coding agent

Spawn coding-agent CLI sessions, review tool calls inline, approve permission prompts, and steer in-progress work — all from your phone.

Git workflows on mobile

Browse status, hunk-by-hunk diffs, stage, commit, push — with the same keystrokes you'd use at your desk.

A PTY terminal in your pocket

Real PTY-backed shells with persistent scrollback. Tail logs, kick off builds, attach to a tmux session, jump back in mid-stream — the screen redraws without disturbing the running process.

Quick start

Three steps: install a coding-agent CLI, install Quicksave, then pair from your phone.

  1. 1

    Install a coding-agent CLI and sign in

    Pick whichever you prefer — Quicksave will drive whichever one is on your $PATH.

    Claude Code
    Official setup ↗ 
    curl -fsSL https://claude.ai/install.sh | bash
claude  # follow login prompts
    Codex
    Official setup ↗ 
    npm install -g @openai/codex
codex  # follow login prompts
  2. 2

    Install the Quicksave agent on your dev machine

    Needs Node.js 20+ and git. Full notes in the agent README. 

    npm install -g @sumicom/quicksave
cd /path/to/your/repo
quicksave

    Prints a pairing URL and QR code. Keeps a background daemon running.

  3. 3

    Pair from your phone

    Open localhost and scan the QR code. Add to home screen for the full PWA experience. Everything from this point is end-to-end encrypted.

Self-host the relay

Don't want to depend on the public relay? Build the Docker image and point your PWA at it. 

docker build -f apps/relay/Dockerfile -t quicksave-relay .
docker run -p 8080:8080 quicksave-relay

Full deployment checklist in docs/relay/deployment.md.
